# Results

This directory contains the output files generated by the pipeline.

## Directory Structure

```
results/
├── plots/          # Generated visualizations
│   ├── feature_importance.png
│   ├── quantum_circuit.png
│   ├── kernel_matrices_heatmap.png
│   ├── roc_curves.png
│   └── confusion_matrix.png
│
└── logs/           # JSON result files
    ├── selected_features.json
    ├── hai_selected_features.json
    ├── final_benchmark.json
    ├── final_robustness_stats.json
    └── real_hardware_validation.json
```

## Log File Descriptions

### selected_features.json / hai_selected_features.json

Contains the feature selection results including:
- Selected feature names and their importance scores
- Qubit assignments for the quantum circuit
- Random forest parameters used
- Total importance coverage

### final_benchmark.json

Contains the benchmark comparison results:
- QSVM metrics (F1, AUC, Accuracy, Precision, Recall)
- Classical SVM metrics (RBF kernel)
- Improvement percentages

### final_robustness_stats.json

Contains multi-seed validation results:
- Results for each random seed (42, 123, 999, 2024, 555)
- Mean and standard deviation across seeds
- Per-dataset breakdowns

### real_hardware_validation.json

Contains IBM Quantum hardware execution results:
- Job IDs for verification
- Transpiled circuit metrics (depth, gate counts)
- Backend information (ibm_fez)
- Execution timestamps

## Regenerating Results

All files in this directory can be regenerated by running the pipeline:

```bash
# Feature selection results
python src/swat_feature_selection.py
python src/hai_feature_selection.py

# Benchmark results
python src/statistical_robustness_suite.py

# Hardware validation
python src/real_hardware_validation.py
```

Note that hardware validation requires an IBM Quantum account.
